üí° Implementation Approach

Add a new workflow tag to assets (e.g., workflow:discovery, workflow:demo). Then configure role hub pages to section by workflow tags instead of format tags. No schema changes needed ‚Äî just new tag values and hub page configuration.

New Tag Conventions
workflow:*
Tag prefix
Task-based grouping: workflow:discovery, workflow:demo, workflow:competitive

Tag prefix

Tag prefix
Customer association: customer:allbirds, customer:rothys (links metrics to examples)

üí° Why Tag Conventions Over Schema?

Using tag prefixes (workflow:, competitor:, customer:) lets you add rich metadata without CMS schema changes. Your existing tag system already supports this ‚Äî it's just about establishing conventions.

6

Implementation Steps

Phased rollout starting with highest-impact changes.

1

Define Workflow Tags (Day 1)

Create the workflow:* tag taxonomy. Start with 8-10 core workflows covering Sales and CSM journeys. Document in your tag guidelines.

2

Auto-Tag Existing Assets (Day 2-3)

Write a migration script to auto-assign workflow tags based on existing tags and types. Review output and adjust mappings.

3

Update Sales Hub Sections (Day 4)

Configure /hub/sales to section by workflow tags instead of format. Test navigation flow and adjust asset assignments.

4

Update CSM Hub Sections (Day 5)

Apply same pattern to /hub/csm. Validate with CSM team for relevance and completeness.

5

Build Cross-Link Component (Week 2)

Create reusable "Related Assets" sidebar component. Implement auto-linking logic based on shared tags. Add to asset detail page template.

6

Add Proof Point Integration (Week 2)

Build "Supporting Proof Points" panel for Content and Competitive assets. Query proof hub by matching competitor/product tags.

7

Add Training Recommendations (Week 3)

Build "Recommended Training" panel. Link Content/CoE assets to relevant Enablement hub content based on topic overlap.

8

Expand to SC & BDR Hubs (Week 3-4)

Define workflow taxonomy for SC (Technical Discovery, Integration, Custom Solutions) and BDR (Prospecting, Outreach, Qualification). Populate hubs.
