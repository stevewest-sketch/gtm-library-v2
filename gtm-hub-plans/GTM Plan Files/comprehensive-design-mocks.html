Section 1

Responsive Card Fix

Addresses the badge/format cutoff issue you showed. Key changes: truncation, smaller fonts, better overflow handling.

Problem: Badge Overflow

  • Long type badges like "PRODUCT TRAINING" get cut off
  • Format pill ("Live Replay") overflows card edge
  • No graceful degradation at narrow widths

Solution: Responsive Badges

  • max-width: 140px on type badge with ellipsis
  • Smaller font sizes: 10px badge, 11px format
  • Format hidden on very narrow cards (@media query)
  • Left accent bar instead of top border (better for narrow cards)
  • Card height reduced to 200px (from 220px)

Implementation Notes

CSS/Component Changes Summary

Responsive Card Changes (AssetCard.tsx)

  • Add max-width: 140px to type badge with text-overflow: ellipsis
  • Reduce font sizes: badge 10px, format 10px, title 14px
  • Add @media (max-width: 400px) to hide format on very narrow cards
  • Consider reducing card height from 220px to 200px
  • Move accent from top border to left border (optional but helps narrow cards)

Home/Library Landing Changes

  • Add hero section with stats to app/page.tsx
  • Keep boards grid from LibraryBrowseContent.tsx
  • Add "Quick Access" cards section
  • Add "Recently Added" section with 3-card preview
  • Consider merging / and /library routes

Asset Detail Improvements

  • Create two variants: simple (single column) vs training (two column with sidebar)
  • Add preview card with "Open Asset" button
  • Improve metadata grid layout (4 columns, responsive to 2)
  • Better tag styling with hover states
  • For training: add takeaways, tips section, session materials sidebar
